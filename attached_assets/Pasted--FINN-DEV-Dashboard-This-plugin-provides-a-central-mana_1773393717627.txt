# FINN DEV Dashboard

This plugin provides a central management interface for internal Finn Partners WordPress plugins. It handles automatic updates via the GitHub API and manages site licensing through a central registry.

## 1. Configuring the Dashboard (One-Time Setup)

The **App ID** and **Installation ID** are hardcoded. To authenticate a client site, you simply upload the GitHub App credentials.

### How to Add the Private Key
1. **Retrieve the Key:** Download the `.pem` file from LastPass Shared Dev Team vault (e.g., `finn-dev-dashboard.2026-02-25.private-key.pem`).
2. **Access Settings:** In the WordPress admin, navigate to **FINN DEV > Settings**.
3. **Upload:** Under the **Authentication** panel, click "Choose File" and select your `.pem` file.
4. **Connect:** Click **Connect Dashboard**. The system will extract the key and store it securely.

### Managing the Connection
- **Removing Access:** You can click **Remove Private Key** at any time to disconnect the site from the GitHub API.
- **Replacing Keys:** If a key expires, use the **Replace Existing Key** upload tool in the same settings area.

> **Note:** Access to the FINN DEV menu is restricted to users logged in with a `@finnpartners.com` email address.

## 2. Enabling Updates for a Custom Plugin

To connect a new plugin to the dashboard, you must add a specific header and manage releases correctly.

### Step A: Add the GitHub Repo Header
Add the `GitHub Repo` line to your plugin's main PHP file header. The syntax must be exact:

```php
/**
 * Plugin Name: FINN RSS
 * Version: 1.0.0
 * GitHub Repo: finnpartners/fp-rss
 */
```

### Step B: Grant Repository Access
Ensure the GitHub App has permission to access the specific repository in the GitHub Organization settings.

### Step C: Deploy an Update
The dashboard relies on Git Tags for updates:

1. Bump Version: Update the Version in your .php file (e.g., to 1.0.1).
2. Push: Commit and push changes to the main branch.
3. Release: Create a new GitHub Release with a tag (e.g., v1.0.1) that matches the file version.

## 3. Licensing & Registry
The dashboard pings the FP License Registry daily to verify the site's status.

- Automatic Exclusion: The fp-license-registry repository is automatically hidden from the "Available Plugins" list to keep the UI clean.
- Kill Switch: If a domain is marked as revoked in the central licenses.json, all internal FINN plugins on that site will be automatically deactivated.
- Heartbeat: The site sends a "heartbeat" to GitHub whenever plugins are activated or checked, keeping the registry up to date with the current environment.
